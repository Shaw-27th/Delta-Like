# 新 Demo 设计文档

## 文档规则

- 这份文档用于给用户查看当前设计与实现进度。
- 英文版文档是维护源，保持英文。
- 中文版文档是镜像文档，每次英文版有实质更新后都应刷新。
- 文档内容应反映当前真实可玩的状态，而不是已经过时的计划。

## 项目概要

- 这是一个以单局突袭为核心的战术探索原型。
- 玩家在局外据点整备，进入固定地图，在房间中实时清敌、搜索、搜刮，并尝试撤离带出战利品。
- 当前主入口链路是 `project.godot -> NodeRaidDemo.tscn -> RaidMapDemo.cs`。

## 当前核心循环

1. 在据点整备。
2. 购买、出售、招募。
3. 选择固定地图模板与难度。
4. 进入单局。
5. 在房间视图中沿房间连接推进，战略地图作为浮窗辅助。
6. 战斗、搜索、搜刮、管理携行空间，再决定继续还是撤退。
7. 成功撤离或行动失败。
8. 返回据点状态。

## 当前已实现状态

### 地图与移动

- 当前使用固定地图模板，不是程序生成拓扑。
- 房间之间由明确节点连接。
- 战略地图浮窗会随当前视口映射缩放。
- 底板地图层不显示地名。
- 地名只显示在节点层。
- 房间出入口使用连续方向映射。
- 进出房间按真实连线方向匹配，而不是四方向硬吸附。
- 进入下一个房间时，会从上一个出口方向的反向进入，因此往返方向一致。

### 房间玩法

- 房间视图已经是主要玩法层。
- 战斗、搜刮、容器互动、房间移动都发生在同一房间状态里。
- 战略地图更多是规划层，不是主要交互面。
- 当前房间没有敌人时，主控英雄探索移速翻倍。

### 战斗与反馈

- 房间战斗层支持英雄、我方士兵、怪物和 AI 小队。
- 近战与远程单位已有明显区分。
- 命中、硬直、击退循环已经接通。
- 近战弧光效果已经改成弯曲斩击反馈。
- 已有程序化运行时音效：
- 轻近战攻击
- 重近战攻击
- 轻远程攻击
- 重远程攻击
- 轻受击
- 重受击
- 死亡
- 近战冲击三角退化时已经做保护，避免 Godot 三角剖分报错。

### AI 表现

- AI 小队使用橙黄暖色系。
- 普通怪物和威胁单位使用灰白色系。
- 死亡 X 标记继承单位本身的颜色体系。

### 容器与搜索

- 容器不再以右侧边栏为主交互方式。
- 容器直接布置在房间场景中，并可在非战斗时近距离自动打开。
- 搜索当前只允许在非战斗状态进行。
- 现有揭示 / 搜索 / 拿取逻辑仍基于容器格子系统。
- 搜索面板现在是半透明浮窗，而不是整屏压暗的模态框。
- 搜索面板打开时玩家仍可移动。
- 离开当前容器交互范围会中断搜索并关闭面板。
- 正在进行中的搜索不能手动关闭。
- 容器支持 `全部拾取`，但只拿走当前可拿的物品。

### 死亡掉落

- 普通单位死亡不再稳定生成尸体容器。
- 普通非英雄死亡会掉资源球。
- 资源球当前只提供金钱。
- 主控英雄靠近后会自动吸附并收取资源球。
- 本局拾取到的金钱会在侧栏单独显示。
- 普通敌方单位有小概率额外掉一个特殊容器。
- 我方普通士兵不会掉常规尸体容器。
- 精英与 Boss 仍会单独掉精英容器。
- 背包中丢弃的物品会在房间里形成临时弃置容器。

## 携行容量与队伍背包

### 当前规则

- 局内背包不再是无限列表。
- 搜到的物品会进入真实有限的队伍背包。
- 物品尺寸在容器与背包之间保持一致。
- 如果某物品在容器里是 `2 x 3`，进背包后仍是 `2 x 3`。
- 局内 UI 会显示占用 / 上限。

### 当前容量来源

- 当前没有独立的基础队伍容量块。
- 英雄容量贡献：`2 x 2`
- 普通士兵容量贡献：`1 x 1`
- 容量当前来自仍然存活的我方房间单位。

### 当前复合布局规则

- 容量是由多个容量块拼出来的，不是一个完整矩形背包。
- 当前每行上限是 `12` 格。
- 不同来源容量块会用虚线边界辅助说明。
- 这些可视化边界不构成硬放置限制。
- 无容量的格子不会被绘制出来。
- 背包格子为了测试已经放大。

### 当前背包生长规则

- 容量块按 `4 x 4` 阶段区域增长。
- 系统会优先让当前区域接近方形。
- 同分时按从上到下、从左到右顺序落位。
- 当前 `4 x 4` 区域尽量填满后才会向右扩新区域。
- 这条规则是为了尽量保留 `2 x 3` 这类大件所需的连续空间。

### 当前背包交互

- 已支持拖起物品。
- 已支持旋转。
- 拿起物品时会偏移显示，避免鼠标完全压在物品中心上。
- 已支持悬停显示名称。
- 较大的物品会在可读情况下直接显示名称。
- 已有自动整理按钮。
- 自动整理会同时考虑背包区和待整理区。
- 自动整理时允许自动旋转物品。

### 待整理与超载

- 新物品放不下时会先尝试自动整理。
- 因队友死亡导致容量缩减时，也会先自动整理。
- 仍然放不下的物品会进入右侧 `待整理` 区。
- 只要待整理区有东西，就视为超载。
- 超载时不能过门。
- 超载时不能撤离。
- 超载时仍可在当前房间内移动。

### 当前超载解法

- 待整理物品可以丢回当前房间。
- 正在手持的背包物品也可以丢回当前房间。
- 这些丢弃行为会创建或复用一个 `临时弃置` 容器。
- 这是当前局内解除超载的实际方案。

## 据点与经济

- 据点买卖流程已可用。
- 征募流程已可用。
- 当前征募默认得到 `新兵`。
- 当前有一套默认 roster 种子。
- 为了测试，开局资金被提高。
- 当前测试开局资金：`3000`

## 士兵成长快照

### 当前第一波实现

- 士兵已经持久记录兵种与经验。
- 当前已落地兵种集合：
- `新兵`
- `盾兵`
- `精锐盾兵`
- `钢盔盾卫`
- `壁垒盾卫`
- `枪兵`
- `刀兵`
- `弓兵`
- `骑兵`
- 据点里已有基础士兵 roster 面板。
- 新兵可以在局外从这个 roster 面板升阶。
- 升阶当前会消耗金钱并要求已有经验。
- 骑兵仍是第一波基础兵种里成本最高的一档。
- 存活士兵会在清空房间和成功撤离时获得经验。
- 在房间战斗里，兵种会影响：
- 生命
- 速度
- 伤害
- 攻击距离
- 体力
- 远近战行为
- 当前已有最小小兵技能框架：
- 主动技能
- 被动技能
- 对大部分非远程小兵来说，`跑动` 被当作默认主动技能。
- 基础盾兵不跑动，只保留远程减伤被动。
- 当前盾兵线已经是四段成长：
- `盾兵`
- `精锐盾兵`
- `钢盔盾卫`
- `壁垒盾卫`
- `精锐盾兵` 是第一段升级，并拥有基础版 `盾冲`。
- `钢盔盾卫` 增加头盔，并获得概率格挡任意伤害的强化被动。
- `壁垒盾卫` 保留头甲轮廓、使用更华丽的大盾，并拥有强化版 `盾冲` 与路线冲击波特效。
- `盾冲` 消耗体力、冷却 `5s`，会沿冲锋路径撞开单位，并在终点造成更强的控制。
- 盾系被动当前仍保留对远程攻击 `50%` 减伤。
- `Armor` 现在已经是实际参与结算的战斗属性。
- 更高盾阶会在模型上体现为甲、头盔和更强盾牌样式。
- 骑兵当前仍有比一般线列步兵更重的冲击型攻击轮廓。

## 库存进展快照

### 已完成：仓库基础闭环

- 仓库已经变成真实固定网格，而不是字符串列表。
- 当前仓库尺寸为 `12 x 10`。
- 商店买入会尝试把物品放入真实仓库网格。
- 据点仓库 UI 已经是网格显示。
- 仓库物品可选中并出售。
- 仓库物品也可在仓库内部重新拖放整理。

### 进行中：统一库存转移逻辑

- 商店与仓库已经共享：
- 物品尺寸语义
- 稀有度颜色显示
- 网格化物品表现
- 据点仓库与局外背包之间已有双向拖放转移。
- 据点与结算页都支持按下、移动、松手的拖放流程，而不只是两次点击拿起 / 放下。
- 拖拽预览现在会跟随抓取偏移，放置判断也和预览左上角统一。
- 据点布局已经重排，仓库、商店、局外背包各自占独立框体。
- 旧的 `带入行动 / 放回仓库` 按钮已不再是主流程。
- 现在默认是仓库与局外背包直接网格搬运。
- 拖着的局外物品会始终画在最上层，不会被后画的 UI 压住。
- 容器到背包的转移已经接好。
- 背包放回当前打开容器的双向转移也已接好。
- 放回容器时会先尝试点击格，再尝试自动找位，必要时还会尝试旋转后落位。
- 容器弹窗在你手持背包物品时也会显示明确提示。
- 撤离成功后，物品不会自动直接塞进仓库。
- 而是先回到持久性的局外背包。
- 结果页后会进入结算转移页，再由玩家把物品从局外背包转进仓库。
- 结算页已有 `整理背包` 和 `全部入仓` 快捷操作。

### 当前库存完整目标

- 当前目标不是单纯让几个面板都能用。
- 目标是让以下系统共享一套库存语言：
- 房间容器
- 队伍背包
- 仓库
- 商店获取路径
- 理想结果是：
- 相同的物品尺寸规则
- 相同的物品表现逻辑
- 相同的转移预期
- 更少的面板特例

- 等级

- 装备成长框架

### 业务模块

- 后续如果继续扩系统，仍然需要继续分层整理。
